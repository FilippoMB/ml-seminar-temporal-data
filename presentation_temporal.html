Deep Learning for Temporal Data

From fixed windows and recurrent models to reservoir computing, temporal convolutions, transformers, and remote-sensing time series.

30 minutes 20-25 slides technical introduction
Motivation

Temporal data asks for memory

We observe a process through ordered measurements and want a model that remembers the right past at the right scale.

  • Forecast: predict future values.
  • Classify: assign a label to a sequence.
  • Detect: find changes, events, or anomalies.
  • Impute: reconstruct missing observations.

Formal setup

A minimal sequence-learning view

\[ \mathcal{F}_{\theta}\left(x_{t-W:t}, u_{t:t+H}\right) = \hat{x}_{t:t+H} \]
  • \(W\): context length, or how much history the model sees.
  • \(H\): prediction horizon.
  • \(u_t\): exogenous information such as calendar, weather, orbit, or metadata.

Architectural question:

How should information move across time?

Learning across collections

Local and global predictors

Local models

One model per signal: tailored, but data inefficient.

\(\hat{x}^i_{t+h}=f(x^i_{t-W:t};\theta^i)\)

Global models

One shared model: sample efficient and scalable.

\(\hat{x}^i_{t+h}=f(x^i_{t-W:t};\theta)\)

Fixed windows

A sliding context becomes a table

Choose a fixed window of \(P\) past values and learn a map to the target horizon.

\[ \hat{x}_{t+\tau}=g\left([x_{t-P},\ldots,x_{t-1},x_t]\right) \]
  • Linear AR: \(g\) is linear.
  • Tree/SVM/MLP: \(g\) can be nonlinear.
  • Every new time step shifts the window.
Nonlinear windows

An MLP is a learned window operator

  • Input dimension: \(d_{\mathrm{in}}=P\).
  • Output dimension: \(d_{\mathrm{out}}=H\), or a single horizon.
  • Nonlinear layers learn interactions among lags.
  • Training pairs are made by shifting a long sequence into many past/future examples.
Notebook example

Electricity demand: nonlinear and seasonal

The temperature/load relation is often U-shaped rather than linear.

  • Demand depends on temperature, calendar, and daily/weekly rhythms.
  • A useful baseline is seasonal persistence: tomorrow resembles yesterday.
  • Neural models must beat simple temporal baselines before we trust the extra complexity.
Limitation

What should \(P\) be?

  • Too short: the relevant evidence is outside the window.
  • Too long: more parameters, more noise, weaker local resolution.
  • A fixed window is a hard architectural assumption about memory.

This is the doorway to recurrent states, reservoirs, convolutions, and attention.

RNN idea

A hidden state carries context

  • Process one observation at a time.
  • Update a hidden state \(h_t\) with the new input and old state.
  • Use \(h_t\) as a compressed summary of the past.
\[ h_t=\phi(W_x x_t + W_h h_{t-1}), \qquad y_t=W_o h_t \]

Training

Backpropagation through time

  • Unroll the recurrence into a deep feed-forward graph.
  • Share parameters across all time steps.
  • Gradients must travel through many repeated transformations.

Long memory is possible, but optimization can become the bottleneck.

Failure modes

Why vanilla RNNs are hard to scale

Vanishing gradients

Distant events receive tiny updates and are forgotten during training.

Exploding gradients

Repeated dynamics can amplify errors; clipping helps but does not solve modeling.

Sequential compute

Time steps depend on previous hidden states, limiting parallelism.

Gated units such as LSTMs and GRUs improve memory control, but the recurrence is still sequential.

Echo State Networks

A randomized RNN with a trained readout

  • Random input and recurrent weights create a high-dimensional dynamical system.
  • The reservoir state \(h_t\) contains many nonlinear temporal features.
  • Only the output map \(W_o\) is fitted, often with ridge regression.
Readout

One reservoir, many forecasting horizons

Readout selects features for horizon \(\tau_1\).

A different readout can select features for horizon \(\tau_2\).

\[ x_{t+\tau}=g(h_t), \qquad h_t=\tanh(W_i x_t + W_h h_{t-1}) \]
Reservoir tuning

Memory lives near the edge of chaos

  • Spectral radius \(\rho\): controls contraction and memory.
  • Input scaling: controls the linear/nonlinear operating regime.
  • Reservoir size: trades richer dynamics for computation.

Reservoirs trade supervised training for a rich, redundant pool of dynamical features.

Convolution

A filter is a sliding learned window

A 1-D convolution applies the same small kernel at every time position.

\[ z_t=\sum_{k=0}^{K-1} w_k x_{t-k} \]
  • Like AR: local lagged features.
  • Unlike AR: many nonlinear channels stacked in depth.
  • Unlike RNN: all positions can be processed in parallel.
Temporal convolutional networks

Depth and dilation expand the receptive field

  • Causal padding prevents future leakage.
  • Dilation skips positions to cover longer history.
  • Residual blocks make deep temporal filters trainable.

A TCN keeps the window intuition while escaping one fixed, shallow window size.

Analogy

Classic windows vs temporal convolutions

Classic window model

  • Explicit lag vector.
  • One predictor sees the full chosen window.
  • The window length is an input-design decision.

1-D convolutional model

  • Small learned filters scan the series.
  • Depth composes local temporal motifs.
  • The receptive field grows by architecture.

See Bai, Kolter, and Koltun, 2018, for a broad empirical comparison of TCNs and recurrent networks.

Self-attention

Content-addressable memory over time

\[ \mathrm{Attn}(Q,K,V) =\mathrm{softmax}\left(\frac{QK^\top}{\sqrt{d}}\right)V \]
  • Each time step produces a query, key, and value.
  • Attention weights decide which other steps matter.
  • Multi-head attention learns several relation types at once.
Tradeoffs

Why transformers changed sequence modeling

Strengths

  • Direct long-range interactions.
  • High parallelism during training.
  • Flexible conditioning on positional encodings, calendar features, metadata, and masks.

Costs

  • Quadratic attention in sequence length.
  • Less built-in locality than convolutions.
  • Data hungry without good inductive bias.

Transformer architecture: Vaswani et al., 2017.

  • Labels can be pixel-level, parcel-level, or region-level.
  • Architecture choice decides how spectral, spatial, and temporal cues interact.

Takeaways

Same question, different memory mechanisms

State

RNNs and reservoirs compress the past into a moving representation.

Filter

Temporal convolutions learn local windows and compose them into large receptive fields.

Attention

Transformers select useful time steps directly, especially with masks and metadata.

For temporal data, architecture is a theory of what kind of past matters.
