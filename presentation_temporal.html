Reservoir computing

An Echo State Network trains only the readout

  • Random recurrent dynamics
  • Rich temporal features
  • Train only the readout
\[ \begin{aligned} h_t&=\tanh(W_i x_t + W_h h_{t-1})\\ \hat{y}_{t+\tau}&=W_o h_t \end{aligned} \]
Reservoir practice

Useful when training recurrence is expensive

  • Spectral radius: memory
  • Input scaling: nonlinearity
  • Ridge readout: overfitting control

Useful when recurrent training is too expensive.

Convolution

A filter is a sliding learned window

Apply the same small temporal kernel everywhere.

\[ z_t=\sum_{k=0}^{K-1} w_k x_{t-k} \]
  • Local lag features
  • Stacked nonlinear channels

Temporal convolutional networks

Depth and dilation expand the receptive field

  • Local filters compose
  • Dilation skips positions
  • Depth grows context

Receptive field is about how much history can influence a representation.

Causal convolution

Causal padding keeps the future hidden

Architecture map

Which memory mechanism fits which problem?

Architecture Memory mechanism Parallelism Good fit Watch out
Window + MLP Fixed explicit lags High Baselines; engineered features Choosing \(P\)
RNN/GRU/LSTM Learned recurrent state Low across time Streaming; latent stages Hard long training
Reservoir/ESN Random dynamical features Medium Limited labels; fast readouts Tuning sensitive
TCN Receptive field by depth/dilation High Phenology; local motifs Context by design
Transformer Direct attention over observed dates High Irregular dates; masks Memory cost
SSM/Mamba State space scan with selective memory High / streaming Long sequences; efficient context Newer ecosystem
