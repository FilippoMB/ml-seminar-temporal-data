Motivation

Temporal data is about usable memory

The signal is often not in one observation, but in how observations change.

  • Horizon: what future or label?
  • Order: which past evidence matters?
  • Missingness: gaps, clouds, irregular dates
  • Leakage: keep future information hidden

Past context is useful only relative to a prediction horizon.

Limitation

Can the model learn its own memory?

Recurrent Neural Networks

A hidden state carries context

Read \(x_t\) consume the next observation in the sequence
Update \(h_t\) combine current input with the previous memory \(h_{t-1}\)
Predict \(y_t\) decode the updated state into an output

A single update rule is shared across time. The hidden state \(h_t\) is the compact memory passed to the next step.

Training

Backpropagation through time

  • Unroll in time
  • Share weights
  • Backpropagate through the chain

Memory helps, but training can be hard.

Gated recurrence

LSTM gates control memory

Forget gate

Decide which parts of the previous memory survive.

\(f_t = \sigma(W_f[h_{t-1}, x_t] + b_f)\) \(f_t \odot C_{t-1}\)

Input gate

Choose how much new candidate content to write.

\(i_t = \sigma(W_i[h_{t-1}, x_t] + b_i)\) \(\hat{C}_t = \tanh(W_C[h_{t-1}, x_t] + b_C)\) \(i_t \odot \hat{C}_t\)

Cell state and output

Combine retained and written memory, then expose useful state.

\(C_t = f_t \odot C_{t-1} + i_t \odot \hat{C}_t\) \(o_t = \sigma(W_o[h_{t-1}, x_t] + b_o)\) \(h_t = o_t \odot \tanh(C_t)\)
Reservoir computing

Echo State Networks

  • Random recurrent (\(W_h\)) and input (\(W_i\)) weights
  • Train only the readout
  • Large reservoir produces rich temporal features
  • Readout select only the features useful for the task

Useful when recurrent training is too expensive or when data are not enough.

Trade-offs: lack of training make hyperparameters critical

Reservoir configuration

  • Spectral radius: controls the internal dynamics of the Reservoir and the amount of memory
  • Low values: Contractive regime, short memory, no expressivity
  • High values: Chaotic regime, overfits and cannot generalize
1-D convolution

A temporal filter scans for reusable motifs

Image analogy

A small kernel scans space and reuses the same detector at every location.

Temporal filter

The same learned kernel scans time to detect slopes, peaks, pulses, or changes.

The same learned kernel is applied at every time step.

\[ z_t=\sum_{k=0}^{K-1} w_k\,x_{t-k} \]
  • Weight sharing: same detector everywhere
  • Feature map: response over time
  • Filter bank: many motifs in parallel
The animation is intentionally similar to a sliding window, but the message is different. A windowed MLP flattens a chosen context and learns position-specific interactions. A convolution learns a local detector and reuses it across time. Multiple filters create multiple feature maps, like channels in an image CNN.
Temporal convolutional networks

Stacked filters build long temporal features

A TCN grows memory by composing small filters across layers.

\[ R = 1+\sum_{\ell=1}^{L}(K_\ell-1)d_\ell \]
  • Layer 1: local slopes and pulses
  • Deeper layers: longer temporal patterns
  • Dilation: wider context without dense kernels

The receptive field is the maximum history that can affect one output.

Causal convolution

Causal padding keeps the future hidden

For forecasting, every output must ignore observations from the future.

  • Left padding: preserve sequence length
  • Right alignment: output belongs to current time
  • No symmetric context: future values are forbidden

Causality is not just a layer option: preprocessing, normalization, and imputation must also respect time.

Phenology example

Temporal filters detect phenology and change

A temporal CNN acts like a learned bank of matched filters.

  • Green-up: increasing vegetation signal
  • Peak / plateau: crop calendar evidence
  • Drop: harvest, flood, burn, disturbance

Good fit when local temporal shapes are informative and transferable across parcels.

Example temporal motifs

Self-attention

Attention retrieves from the observed past

Each token asks which other observations are relevant.

\[ \mathrm{Attn}(Q,K,V;M) = \mathrm{softmax} \left( \frac{QK^\top}{\sqrt{d}} + M \right)V \]
  • Query/key: score relevance
  • Value: information to retrieve
  • Mask: hide clouds or future steps

Attention selects observations by content; time encodings tell it when they occurred.

Spend little time on the full Transformer architecture. The important idea here is retrieval. A token computes scores against other tokens, normalizes those scores, and combines the values. For forecasting, the mask must remove future observations. For sequence classification, the model may be allowed to attend bidirectionally over the observed sequence.
Transformers

What changes relative to windows, RNNs, and TCNs?

A Transformer does not scan or compress the sequence first: it builds direct token-to-token links.

Model How it uses the past Main limitation
MLP window Dense map from fixed lags Choose \(W\); lag positions are fixed
RNN / LSTM Compress past into \(h_t\) Sequential bottleneck; long credit paths
TCN Apply local filters over a receptive field Context set by depth and dilation
Transformer Retrieve directly from allowed tokens \(T^2\) attention; needs time and masks

Best fit: non-local dependencies, irregular observations, and tasks where different dates matter for different examples.

Transformer architecture: Vaswani et al., 2017.

This is the main comparison slide. Do not say Transformers are always better. Say that they change the inductive bias. MLPs assume a fixed window, RNNs assume the past can be summarized in a state, and TCNs assume useful temporal motifs can be detected locally and composed. Attention assumes that useful context can be selected by learned pairwise relevance. The price is the pairwise attention matrix and the need to encode time/order explicitly.
Irregular data

Tokens need time and validity

Each observation becomes a token: what was measured, when it was observed, and whether attention may use it.

  • Signal: bands, indices, image features
  • Time: date, day-of-year, \(\Delta t\)
  • Mask: Unavailable regions and future information
